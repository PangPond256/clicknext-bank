# Clicknext Bank API

Clicknext Bank API คือระบบหลังบ้านจำลองการทำงานของธนาคาร พัฒนาด้วย ASP.NET Core Web API รองรับระบบสมาชิกและการทำธุรกรรมทางการเงินพื้นฐานผ่านเครือข่ายอินเทอร์เน็ต

ข้อควรทราบก่อนใช้งาน (Important Note)
--------------------------------------------------

เนื่องจากระบบส่วนหน้า (Frontend) ถูกติดตั้งบน Vercel ซึ่งใช้โปรโตคอล HTTPS ส่วนระบบหลังบ้าน (Backend) ถูกติดตั้งบน Somee ซึ่งใช้โปรโตคอล HTTP ธรรมดา

เบราว์เซอร์จะบล็อกการเชื่อมต่อโดยอัตโนมัติเนื่องจากนโยบายความปลอดภัย (Mixed Content) เพื่อให้ระบบทำงานได้ถูกต้อง **แนะนำให้ตรวจสอบผลงานผ่าน Google Chrome บนคอมพิวเตอร์** โดยตั้งค่าดังนี้:

### สำหรับการใช้งานบนคอมพิวเตอร์ (Desktop)
1. เปิดหน้าเว็บ Vercel ของโปรเจกต์นี้
2. คลิกที่สัญลักษณ์แม่กุญแจ หรือรูปวงกลม (i) หน้าแถบที่อยู่เว็บ (Address Bar)
3. เลือก **การตั้งค่าเว็บไซต์ (Site Settings)**
4. ค้นหาหัวข้อ **เนื้อหาที่ไม่ปลอดภัย (Insecure content)** และเปลี่ยนค่าเป็น **อนุญาต (Allow)**
5. กลับมาที่หน้าเว็บแล้วทำการรีเฟรช (Refresh) หนึ่งครั้ง

### สำหรับการใช้งานบนมือถือ (Mobile - Chrome/Android)
1. เปิดหน้าเว็บผ่าน Google Chrome
2. แตะที่ไอคอนแม่กุญแจ หรือไอคอนวงกลม (i) หน้าแถบ URL
3. เลือก **การตั้งค่าเว็บไซต์ (Site settings)**
4. ค้นหาเมนู **เนื้อหาที่ไม่ปลอดภัย (Insecure content)** แล้วเปลี่ยนเป็น **อนุญาต (Allow)**
5. ทำการรีเฟรชหน้าเว็บอีกครั้ง
*(หมายเหตุ: บนระบบ iOS/Safari อาจมีการบล็อกการเชื่อมต่อ HTTPS ไปยัง HTTP อย่างเข้มงวด แนะนำให้ทดสอบผ่านคอมพิวเตอร์เพื่อผลลัพธ์ที่แม่นยำที่สุด)*

--------------------------------------------------
ขอบเขตการทำงานของระบบ
--------------------------------------------------

1. **ระบบยืนยันตัวตน:** สมัครสมาชิกและเข้าสู่ระบบด้วย JWT Token
2. **ข้อมูลผู้ใช้:** ตรวจสอบข้อมูลส่วนตัวของผู้ที่กำลังล็อกอิน (Get Me)
3. **ระบบธุรกรรม:** ฝากเงิน (Deposit) และถอนเงิน (Withdraw) พร้อมคำนวณยอดเงินคงเหลือ
4. **ประวัติการเงิน:** เรียกดูรายการธุรกรรมทั้งหมดและยอดเงินคงเหลือปัจจุบัน (Balance)
5. **การจัดการข้อมูล:** รองรับการแก้ไขและลบรายการธุรกรรม (สำหรับวัตถุประสงค์ในการทดสอบ)

--------------------------------------------------
เทคโนโลยีที่ใช้ (Tech Stack)
--------------------------------------------------

- **Framework:** ASP.NET Core 8.0 Web API
- **Database:** SQL Server (MSSQL) บน Somee
- **ORM:** Entity Framework Core
- **Authentication:** JWT Bearer Token & BCrypt Password Hashing
- **Frontend:** Vue.js (ติดตั้งบน Vercel)
- **API Documentation:** Swagger / OpenAPI

--------------------------------------------------
โครงสร้างโปรเจกต์
--------------------------------------------------

BankApi/
- **Controllers:** จัดการเส้นทาง API และตรรกะการรับส่งข้อมูล (Auth, Transactions)
- **Data:** ส่วนเชื่อมต่อฐานข้อมูล (AppDbContext)
- **DTOs:** รูปแบบข้อมูลสำหรับการรับ Request และส่ง Response
- **Models:** โครงสร้างข้อมูลหลัก (User และ Transaction)
- **appsettings.json:** ไฟล์ตั้งค่าระบบ (Connection String, JWT Key, CORS)
- **Program.cs:** การตั้งค่า Service, Middleware, Authentication และ CORS

--------------------------------------------------
หลักการทำงานและการเชื่อมต่อ (Front to Back)
--------------------------------------------------

ระบบทำงานโดยใช้หลักการแยกระบบส่วนหน้าและส่วนหลังออกจากกัน (Decoupled Architecture)

1. **การพิสูจน์ตัวตน:** เมื่อผู้ใช้ Login สำเร็จ API จะสร้าง JSON Web Token (JWT) ส่งกลับไปให้ส่วนหน้าเก็บไว้ในสถานะของแอปพลิเคชัน
2. **การส่งคำขอที่ปลอดภัย:** ทุกครั้งที่มีการเรียกข้อมูลส่วนตัวหรือทำธุรกรรม ส่วนหน้าจะแนบ Token ไปใน Header (`Authorization: Bearer <Token>`)
3. **การจัดการความปลอดภัย (CORS):** ระบบหลังบ้านมีการตั้งค่า CORS Policy เพื่ออนุญาตให้เฉพาะหน้าเว็บที่กำหนด (Vercel) สามารถสื่อสารกับ API ได้

--------------------------------------------------
คำแนะนำการใช้งาน Swagger
--------------------------------------------------

ในการทดสอบ API ผ่านหน้า Swagger มีขั้นตอนสำคัญในการใส่ Token ดังนี้:

1. ทำการ Login ผ่าน `POST /api/Auth/login` และคัดลอกค่า Token ใน Response
2. คลิกปุ่ม **Authorize** ที่มุมบนขวาของหน้า Swagger
3. ในช่อง Value ให้วาง **เฉพาะค่า Token** เท่านั้น (ไม่ต้องใส่คำว่า Bearer นำหน้า)
4. กดปุ่ม Authorize และทำการเรียกใช้ Endpoint ที่มีการป้องกัน (ที่มีรูปกุญแจ)

*สาเหตุ: ระบบถูกตั้งค่า Bearer Format ไว้ในโค้ดแล้ว Swagger จะเติมคำว่า "Bearer " ให้โดยอัตโนมัติ หากใส่ซ้ำจะทำให้ Token ไม่ถูกต้อง (401 Unauthorized)*
