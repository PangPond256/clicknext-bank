# Clicknext Bank API

Clicknext Bank API คือระบบหลังบ้านจำลองการทำงานของธนาคาร พัฒนาด้วย ASP.NET Core Web API รองรับระบบสมาชิกและการทำธุรกรรมทางการเงินพื้นฐานผ่านเครือข่ายอินเทอร์เน็ต

ข้อจำกัดสำคัญในการทดสอบ (Important Technical Note)
--------------------------------------------------

### การทดสอบผ่านเบราว์เซอร์บนคอมพิวเตอร์ (แนะนำ)
เนื่องจากระบบส่วนหน้า (Frontend) ใช้ HTTPS และระบบหลังบ้าน (Backend) ใช้ HTTP ธรรมดา (Mixed Content) เบราว์เซอร์จะบล็อกการเชื่อมต่อโดยอัตโนมัติ ผู้ตรวจสอบจำเป็นต้องทำการตั้งค่าดังนี้เพื่อให้ระบบทำงานได้:

1. คลิกที่ไอคอนแม่กุญแจหน้าแถบ URL
2. เลือก **Site Settings (การตั้งค่าเว็บไซต์)**
3. หาหัวข้อ **Insecure content (เนื้อหาที่ไม่ปลอดภัย)** และเปลี่ยนเป็น **Allow (อนุญาต)**
4. ทำการรีเฟรชหน้าเว็บหนึ่งครั้ง

### ข้อจำกัดในการทดสอบผ่านสมาร์ทโฟน (Mobile Device)
จากการทดสอบพบว่า **ไม่สามารถใช้งานระบบผ่านเบราว์เซอร์บนมือถือบางรุ่นได้ (เช่น iOS หรือ Android รุ่นใหม่)** เนื่องจากระบบความปลอดภัยระดับ OS บล็อกการเชื่อมต่อ Mixed Content (HTTPS ไปยัง HTTP) อย่างเข้มงวดและไม่สามารถตั้งค่าอนุญาตได้โดยง่าย

**จึงขอแนะนำให้ทำการตรวจสอบและทดสอบระบบผ่าน Google Chrome บนคอมพิวเตอร์เป็นหลักเพื่อให้สามารถเรียกใช้งาน API ได้อย่างสมบูรณ์**

--------------------------------------------------
ขอบเขตการทำงานของระบบ
--------------------------------------------------

1. ระบบยืนยันตัวตน: สมัครสมาชิกและเข้าสู่ระบบด้วย JWT Token
2. ข้อมูลผู้ใช้: ตรวจสอบข้อมูลส่วนตัวของผู้ที่กำลังล็อกอิน
3. ระบบธุรกรรม: ฝากเงิน (Deposit) และถอนเงิน (Withdraw)
4. ประวัติการเงิน: เรียกดูรายการธุรกรรมทั้งหมดและยอดเงินคงเหลือปัจจุบัน
5. การจัดการข้อมูล: รองรับการแก้ไขและลบรายการธุรกรรมเพื่อวัตถุประสงค์ในการทดสอบ

--------------------------------------------------
เทคโนโลยีที่ใช้
--------------------------------------------------

- Framework: ASP.NET Core 8.0 Web API
- Database: SQL Server (MSSQL)
- ORM: Entity Framework Core
- Authentication: JWT Bearer & BCrypt Password Hashing
- Hosting: Somee (API & Database) และ Vercel (Frontend)

--------------------------------------------------
โครงสร้างโปรเจกต์
--------------------------------------------------

BankApi/
- Controllers: จัดการเส้นทาง API และตรรกะการรับส่งข้อมูล
- Data: ส่วนเชื่อมต่อฐานข้อมูล (DbContext)
- DTOs: รูปแบบข้อมูลสำหรับรับและส่งผ่าน API
- Models: โครงสร้างข้อมูลหลัก (User และ Transaction)
- Migrations: ประวัติการเปลี่ยนแปลงโครงสร้างฐานข้อมูล
- appsettings.json: ไฟล์ตั้งค่าระบบ (Connection String, JWT Key, CORS)
- Program.cs: จุดเริ่มต้นของระบบและการตั้งค่า Middleware ทั้งหมด

--------------------------------------------------
หลักการทำงานและการเชื่อมต่อ (Front to Back)
--------------------------------------------------

ระบบทำงานโดยใช้หลักการแยกระบบส่วนหน้าและส่วนหลังออกจากกัน (Decoupled Architecture)

1. การพิสูจน์ตัวตน: ส่วนหน้าส่งข้อมูลเข้าสู่ระบบไปยัง API เมื่อตรวจสอบถูกต้อง API จะสร้าง JSON Web Token (JWT) ส่งกลับไปให้ส่วนหน้าเก็บไว้
2. การส่งคำขอที่ปลอดภัย: ทุกครั้งที่ส่วนหน้าต้องการเรียกข้อมูลส่วนตัวหรือทำธุรกรรม จะต้องแนบ Token ไปในส่วน Header ของ HTTP Request
3. การจัดการความปลอดภัย: ระบบหลังบ้านมีการตั้งค่า Cross-Origin Resource Sharing (CORS) เพื่ออนุญาตให้เฉพาะหน้าเว็บที่กำหนด (Vercel) สามารถส่งข้อมูลเข้ามาใช้งาน API ได้

--------------------------------------------------
คำแนะนำการใช้งาน Swagger อย่างละเอียด
--------------------------------------------------

ในการทดสอบ API ผ่านหน้า Swagger มีขั้นตอนที่ต้องระมัดระวังเป็นพิเศษในส่วนของการใส่ Token:

1. ทำการเข้าสู่ระบบผ่านช่องทาง POST /api/Auth/login เพื่อรับค่า Token
2. คลิกปุ่ม Authorize ที่มุมบนขวาของหน้า Swagger
3. ในช่อง Value ให้วางเฉพาะค่า Token ยาวๆ ที่ได้รับมาเท่านั้น (ไม่ต้องพิมพ์คำว่า Bearer นำหน้า)
4. กดปุ่ม Authorize และ Close

สาเหตุ: เนื่องจากในโค้ดระบบได้ตั้งค่า Security Scheme ให้เป็นรูปแบบ Bearer ไว้แล้ว ตัว Swagger UI จะทำการเติมคำว่า "Bearer " นำหน้า Token ให้โดยอัตโนมัติ หากผู้ใช้ใส่ซ้ำจะทำให้ Token ผิดพลาดและได้รับคำตอบ 401 Unauthorized จากเซิร์ฟเวอร์
